# 手柄测试与校准工具操作说明



## 0. 先确认你要在哪台机器运行

这个软件必须运行在“能直接识别到手柄”的系统里。

- 如果你在 Windows 主机上运行软件：直接把手柄插到 Windows 主机即可。
- 如果你在虚拟机里运行软件：需要把手柄从主机连接到虚拟机，也叫 USB 直通或连接到客户机。

## 2. 创建 conda 环境

不要把依赖装到 base 环境。建议单独建一个环境：


看到命令行前面变成这样，说明环境已经激活。

## 3. 安装软件和依赖

进入项目目录：

```powershell
cd ~\con_test
```

安装本项目：

```powershell
python -m pip install -e .
```

## 4. 第一次启动前检查手柄

插上手柄后，先运行：

```powershell
python -m controller_mapper list
```

你要重点看三件事：

- `[0]`：这是设备编号。只有一个手柄时通常就是 0。
- `Xbox 360 Controller`：这是系统识别到的设备名称。
- `axes/buttons/hats`：这是摇杆轴、按钮、十字键数量。

如果有多个手柄，列表可能是这样：

```text
[0] Xbox 360 Controller
[1] Nintendo Switch Joy-Con
```

后面启动 GUI 时，可以用 `--device` 指定要校准哪一个。

## 5. 启动图形界面

最常用命令：

```powershell
python -m controller_mapper gui
```

如果你有多个手柄，指定第 0 个：

```powershell
python -m controller_mapper gui --device 0
```

启动后窗口里会看到：

- 上方：当前设备编号、设备名称、GUID、当前映射文件。
- 中间：一个通用手柄图。
- 右上：保存和退出按钮。
- 下方：最近一次原始输入，例如 `raw: b0 pressed`。

## 6. 测试按键是否识别正确

打开 GUI 后，直接按真实手柄。

你应该看到：

- 按 A/B/X/Y：界面上对应的 A/B/X/Y 高亮。
- 按 LB/RB：界面上对应肩键高亮。
- 按 LT/RT：界面上对应扳机高亮。
- 按十字键：界面上的箭头高亮。
- 推左右摇杆：界面上的摇杆圆点跟着移动。

判断标准很简单：

- 你按真实 A，界面 A 高亮：正确。
- 你按真实 A，界面 B 高亮：映射错了，需要调整。
- 你推摇杆，界面完全不动：先看底部 `raw:` 有没有变化；如果有变化但图不动，再检查映射。

## 7. 调整错误映射

如果某个键识别不一致，在 GUI 里这样修：

1. 用鼠标点击界面上你想修的目标按钮，例如界面上的 `A`。
2. 软件进入等待绑定状态。
3. 按下真实手柄上应该对应 `A` 的那个物理按钮。
4. 界面会把这个真实输入绑定到 `A`。
5. 继续测试，确认这次按真实 A 时界面 A 高亮。

举例：

- 真实 A 按下后界面 B 高亮。
- 点击界面 `A`。
- 再按真实 A。
- 保存。
- 以后真实 A 就会对应软件里的 `a`。

## 8. 保存映射

保存方式有两种：

```text
点击右上角“保存 S”
```

或者按键盘：

```text
S
```

保存后文件会写到：

```text
~\con_test\mappings
```

如果同名文件已经存在，软件会先生成 `.bak` 备份，再覆盖新文件。

## 9. 退出软件

退出方式：

- 点击右上角“退出 Esc”。
- 或按键盘 `Esc`。

如果当前正在重新绑定某个按钮，第一次按 `Esc` 是取消绑定；没有正在绑定时，`Esc` 才是退出窗口。

## 10. 输出文件在哪

默认输出目录：

```text
~\con_test\mappings
```

每个手柄通常会生成 3 个文件：

```text
<设备名>_<GUID>.json
<设备名>_<GUID>.md
<设备名>_<GUID>.sdl.txt
```

例如：

```text
Xbox_360_Controller_030003f05e0400008e02000000007200.json
Xbox_360_Controller_030003f05e0400008e02000000007200.md
Xbox_360_Controller_030003f05e0400008e02000000007200.sdl.txt
```

## 11. 每个输出文件是干什么的

### `.json`

这是最重要的文件，后续开发主要读它。

它记录：

- 这个手柄叫什么。
- 这个手柄的 GUID 是什么。
- 有多少 axis、button、hat。
- 软件里的标准按钮对应真实手柄的哪个物理输入。

例子：

```json
{
  "controls": {
    "a": "b0",
    "b": "b1",
    "leftx": "a0",
    "dpup": "h0.1"
  }
}
```

含义：

- `"a": "b0"`：软件里的 A 键来自真实手柄第 0 号按钮。
- `"leftx": "a0"`：软件里的左摇杆横向来自第 0 号轴。
- `"dpup": "h0.1"`：软件里的十字键上来自第 0 个 hat 的上方向。

### `.md`

这是给人看的报告文件。

它适合用来检查：

- 哪些标准按钮已经绑定。
- 哪些按钮被跳过。
- 每个按钮对应的真实输入编号是什么。
- 这个手柄的设备信息是什么。

如果你只是想人工核对，打开 `.md` 最舒服。

### `.sdl.txt`

这是给 SDL 生态工具用的映射字符串。

适合给这些方向使用：

- SDL 相关程序。
- pygame/SDL 生态调试。
- Godot 等能理解 SDL gamecontroller mapping 的工具链。

如果你自己的后续开发直接读 JSON，可以暂时不用管 `.sdl.txt`。

## 12. 常用命令汇总

进入项目：

```powershell
cd E:\BaiduSyncdisk\proj\con_test
conda activate controller-map
```

列出手柄：

```powershell
python -m controller_mapper list
```

启动 GUI：

```powershell
python -m controller_mapper gui
```

指定第 0 个手柄启动 GUI：

```powershell
python -m controller_mapper gui --device 0
```

加载指定映射文件启动 GUI：

```powershell
python -m controller_mapper gui --mapping mappings\Xbox_360_Controller_030003f05e0400008e02000000007200.json
```

查看原始输入：

```powershell
python -m controller_mapper monitor
```

验证已保存映射：

```powershell
python -m controller_mapper validate --mapping mappings\Xbox_360_Controller_030003f05e0400008e02000000007200.json
```

导出 SDL 字符串：

```powershell
python -m controller_mapper export-sdl --mapping mappings\Xbox_360_Controller_030003f05e0400008e02000000007200.json
```

## 13. 常见问题

### 运行命令提示没有检测到手柄

按顺序检查：

1. 手柄是否插好或蓝牙是否连接。
2. Windows 控制面板里是否能看到手柄。
3. 如果在虚拟机里，手柄是否已经连接到虚拟机，而不是还连在主机上。
4. 重新运行：

```powershell
python -m controller_mapper list
```

### 命令行前面没有 `(controller-map)`

说明你可能在 base 环境里。

执行：

```powershell
conda activate controller-map
```

再继续运行软件。

### `python -m controller_mapper gui` 启动失败

先确认依赖装在当前 conda 环境里：

```powershell
python -m pip show pygame
```

再确认项目安装过：

```powershell
python -m pip install -e .
```

然后重新启动：

```powershell
python -m controller_mapper gui
```

### 按键高亮和真实手柄不一致

这是正常校准场景，不一定是程序坏了。

处理方法：

1. 点击界面上正确的目标按钮。
2. 按真实手柄上对应的物理按钮。
3. 保存。
4. 再测试一次。

### 输出文件名很长

正常。

文件名里包含设备名和 GUID，是为了区分不同型号、不同系统识别到的手柄。

4. 点击保存。
5. 到 `mappings` 目录查看 `.json`、`.md`、`.sdl.txt`。
